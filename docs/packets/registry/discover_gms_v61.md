### Clientbound adjudications

1. **0x4E (op 78) discovery-internal collision — RESOLVED (CWvsContext wins).**
   `CClientSocket::ProcessPacket` routes the [0x1A,0x5B] window to
   `CWvsContext::OnPacket`, whose case 78 → `sub_830B0B` (OnImitatedNPCData).
   `CNpcPool::OnPacket` (0x5efbad) carries a **dead** 78 case
   (`OnNpcImitateData`) never reached (CField routes only 194-201 to CNpcPool).
   Same dead-case family as v72. Registered at the CWvsContext handler with the
   CNpcPool name as `fname_alt`. The CNpcPool dispatcher was held out of the
   `--apply` union pass to clear the gate; its base ops 194-200 are registered manual.
2. **Char-management 13/14/15 ROTATED SYMBOLS** — the v61 IDB labels on
   `CLogin::OnPacket` cases 13/14/15 are rotated one step off their handler bodies
   (identical to the v72/v79 trap). Registered by the **body-canonical (v83) FName**
   (mirroring the corrected `gms_v72.yaml`): 13=CHAR_NAME_RESPONSE, 14=ADD_NEW_CHAR_ENTRY,
   15=DELETE_CHAR_RESPONSE — verified by decompiled read-order, not the symbol label.
3. **SELECT_CHARACTER_BY_VAC (op 9)** — `CLogin::OnPacket` else-if branch (not a
   switch case). Manual.
4. **CWvsContext 81-100 block ABSENT** — v61 CWvsContext ends at op 91
   (MACRO_SYS_DATA_INIT); the v72 HourChanged→NotifyWedding/Family block (19 ops) is
   absent. The tail 82-91 maps to v72 101-113 by FName.
5. **CField base region compressed** — v61 lacks FIELD_OBSTACLE_ONOFF (single),
   FIELD_OBSTACLE_ALL_RESET, PLAY_JUKEBOX (v72 127/129/131). op 106 = BLOW_WEATHER
   (sub_4ED39C, body Decode4(itemId)+GetItemString+DecodeStr+CHATLOG_ADD).
6. **CUserRemote / CUserLocal shrinkage** — v61 lacks SHOW_UPGRADE_TOMB_EFFECT
   (remote), and MakerResult/KOREAN_EVENT/LOCK_UI/DISABLE_UI/SPAWN_GUIDE (local),
   matching the delta doc. No CHATTEXT1 (single OnChat at op 122).
7. **v61-EXTRA op 21 (0x15)** — ProcessPacket case 0x15 → sub_4747E2 (Decode2 count +
   COutPacket(27) reply); no v72 equivalent, op-name deferred to Stage E.
8. **Dialog-region demuxer ops (239/243/244)** carry provisional op-names
   (ADMIN_SHOP / PLAYER_INTERACTION / MESSENGER) from the v72 structural region;

## Serverbound — method

Read from the v61 `COutPacket(N)` send-sites via a **global COutPacket-ctor harvest**
(ctor `??0COutPacket@@QAE@J@Z` @ `0x5ffc4f`): every code xref to the ctor was walked,
the nearest preceding `push imm` taken as the opcode, and each caller function's
name matched to the `gms_v72.yaml` serverbound FName. **Opcodes were never
region-shift-predicted** — each is the literal COutPacket argument.

The five delta-doc anchors reproduced exactly: LOGIN_PASSWORD=1 (Δ0),
CHANGE_MAP=35 / CHANGE_CHANNEL=36 / MOVE_PLAYER=38 (Δ-2), PARTY_OPERATION=112 /
DENY_PARTY_REQUEST=113 (Δ-10). The shift is **non-uniform**: login≈Δ0 (DELETE_CHAR
drops to the v83-style 23, so PONG/CLIENT_START_ERROR are Δ-1); map/attack region
Δ-2; social/note/keymap region Δ-10; pet/npc/reactor region Δ-19..-25; the
CWvsContext item/ability cluster is Δ0.
