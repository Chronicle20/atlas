### JMS-vs-GMS structural differences (beyond the adjudications)

- **No `CITC::OnPacket` dispatcher.** The CITC class exists, but ITC is
  entered as a stage (`CStage::OnSetITC`, case 124) and its UI drives itself —
  there is no CITC opcode block like GMS v87/v95's 0x170-range MTS ops.
- **No `COpenGatePool::OnPacket` symbol** — the open-gate dispatcher is the
  unnamed sub_5FD3FC with a 3-op cluster (298–300), one more op than v95.
- **No `CBattleRecordMan`, no `CField_KillCount`** (v95-new dispatchers are
  absent from this older-codebase JMS build).
- **Pet block is 8 ops** (173–180, switch tail 0xB0–0xB4 incl.
  `CPet::OnLoadExceptionList` and `CPet::OnActionCommand`) vs GMS v87's 5.
- **ExpeditionIntermediary exists in v185** (absent in GMS until v95) and is
  excluded for the same body-retcode reason.
- JMS-only content ops: pachinko (0x4B dama update; BEANS rows 0x15B–0x15F
  seeded but undispatched in this build), `CCashShop` JMS sub-block
  0x167–0x16A/0x16E/0x16F (unnamed subs).
