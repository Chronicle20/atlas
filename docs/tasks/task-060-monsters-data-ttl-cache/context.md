## 3. Anchor Facts (verified from source)

### 3.1 `monster/information/processor.go` is a 16-line passthrough today

```go
// monster/information/processor.go
func GetById(l logrus.FieldLogger) func(ctx context.Context) func(monsterId uint32) (Model, error) {
    return func(ctx context.Context) func(monsterId uint32) (Model, error) {
        return func(monsterId uint32) (Model, error) {
            return requests.Provider[RestModel, Model](l, ctx)(requestById(monsterId), Extract)()
        }
    }
}
```

The signature **must not change** â€” callers in `processor.go`, `aggro_task.go`,
`recovery_task.go`, etc. take it verbatim.

### 3.2 The REST sentinel is `requests.ErrNotFound`, not a typed `HTTPError`

`libs/atlas-rest/requests/get.go:14-15` exports:

```go
var ErrBadRequest = errors.New("bad request")
var ErrNotFound  = errors.New("not found")
```

`get.go:74-78` returns these directly when the upstream responds 400 or 404.
Any other non-2xx returns `errors.New("unknown error")`. There is **no** typed
`HTTPError` struct.

**Implication for the design:** the error classifier in `cache.go` must use
`errors.Is(err, requests.ErrNotFound)` rather than the typed-error switch shown
in `design.md` Â§4.4. Treat `errors.Is(err, requests.ErrBadRequest)` as transient
(don't cache); everything else is also transient. The negative-cache path
synthesizes the error with `fmt.Errorf("monster %d not found: %w", id, requests.ErrNotFound)`
so `errors.Is(returned, requests.ErrNotFound)` keeps holding for callers.
