## 3. Anchor Facts (verified from source)

### 3.1 `monster/information/processor.go` is a 16-line passthrough today

```go
// monster/information/processor.go (current)
func GetById(l logrus.FieldLogger) func(ctx context.Context) func(monsterId uint32) (Model, error) {
    return func(ctx context.Context) func(monsterId uint32) (Model, error) {
        return func(monsterId uint32) (Model, error) {
            return requests.Provider[RestModel, Model](l, ctx)(requestById(monsterId), Extract)()
        }
    }
}
```

The signature **must not change** — callers in `aggro_task.go`, `recovery_task.go`, `monster/processor.go`, etc., take it verbatim.

### 3.2 The cache value type is `RestModel`, NOT `Model` (Plan-Time Correction)

`information.Model` (`monster/information/model.go:3-19`) has only unexported fields (`hp`, `mp`, `boss`, …). `encoding/json` cannot marshal unexported fields — `json.Marshal(Model{hp: 5})` returns `{}`. `redislib.TenantRegistry` uses `json.Marshal`/`json.Unmarshal` by default (`libs/atlas-redis/tenant_registry.go:30-35`), so a `TenantRegistry[uint32, Model]` would persist `{}` and round-trip every entry to a zero `Model`.

**Correction (already encoded in plan.md):** the positive registry is `TenantRegistry[uint32, RestModel]`. `RestModel` (`rest.go:3-40`) has fully exported, JSON-tagged fields and is the upstream wire format. `processor.go` runs `Extract(rm)` (`rest.go:89-115`) after a positive hit. The cost of `Extract` (a few field copies plus two slice allocations) is sub-microsecond, far cheaper than the Redis roundtrip it sits behind.

design.md §5.2 declares `TenantRegistry[uint32, Model]`; plan.md "Plan-Time Correction" supersedes this declaration.

### 3.3 The REST sentinel is `requests.ErrNotFound`, not a typed `HTTPError`

`libs/atlas-rest/requests/get.go:14-15` exports:

```go
var ErrBadRequest = errors.New("bad request")
var ErrNotFound  = errors.New("not found")
```

`get.go:74-78` returns these directly on HTTP 400 and 404. Any other non-2xx returns `errors.New("unknown error")`. There is **no** typed `HTTPError` struct.

Implication: `classifyError` in `cache.go` uses `errors.Is(err, requests.ErrNotFound)` — sentinel-based, not type-switch. Treat `requests.ErrBadRequest` and every other error as transient (don't cache). `notFoundError` synthesizes `fmt.Errorf("monster %d not found: %w", id, requests.ErrNotFound)` so callers' `errors.Is(returned, requests.ErrNotFound)` checks keep holding for negative-cache hits.

### 3.4 The Redis miss sentinel is `redislib.ErrNotFound` — distinct from `requests.ErrNotFound`

`libs/atlas-redis/tenant_registry.go:48` returns the package-local `ErrNotFound` when `goredis.Nil` is observed. Both packages export an `ErrNotFound` sentinel; they are different errors and are checked in different code paths:

- `errors.Is(err, redislib.ErrNotFound)` → "Redis says no entry; fall through" (after a `posReg.Get`/`negReg.Get`).
- `errors.Is(err, requests.ErrNotFound)` → "upstream HTTP returned 404; record negative entry" (after a `upstreamFn` call) and "synthesize this for negative-cache hits returned to callers".

Mixing them up is the most likely source of an "always-miss" bug.

### 3.5 Two distinct namespaces, one `*goredis.Client`

design.md §2 architecture diagram:

- `atlas:monsters:cache:data:<tenantUUID>:<region>:<major>.<minor>:<id>` — positive (`RestModel` JSON, TTL 5 m default).
- `atlas:monsters:cache:data:not_found:<tenantUUID>:<region>:<major>.<minor>:<id>` — negative (`{}` JSON, TTL 30 s default).

The `atlas:` prefix and tenant key shape come from `libs/atlas-redis/keys.go:27` (`tenantEntityKey`). Two `redislib.NewTenantRegistry` calls share `rc := atlas.Connect(l)` from `main.go:48`. No new client connection.

## 4. Decisions Locked from design.md §6

- **Cache positive value:** `RestModel` (corrected from design.md §5.2's `Model` per §3.2 above).
- **Cache negative value:** `struct{}` (existence is the signal; JSON-marshals to `{}`).
- **Library placement:** none — reuse `libs/atlas-redis.TenantRegistry`.
- **Sweeper:** none — Redis native EX TTL handles eviction.
- **Tenant label:** kept on metrics; documented as cardinality-bounded (≤5 tenants × ≤4 enum values per dimension ≈ 80 series total).
- **Negative TTL default:** 30 s.
- **Error classifier:** sentinel-based (`errors.Is(err, requests.ErrNotFound)`).
- **Master kill-switch:** `MONSTER_DATA_CACHE_ENABLED=false` makes `GetById` skip both registries and behave identically to the pre-task implementation.

## 7. Risk Watch

From `design.md` §11, the risks that need implementer vigilance:

- **JSON-marshalling silent failure on `Model`.** Resolved at plan time — see §3.2 above. Verified: `Model` has unexported fields; the cache must store `RestModel` and `Extract` after hit.
- **Sentinel confusion.** `redislib.ErrNotFound` and `requests.ErrNotFound` are distinct. See §3.4. The wrapper checks the former after Redis ops; emits the latter (via `notFoundError`) on negative-cache hits.
- **Tenant key collision.** Mitigated by using `redislib.NewTenantRegistry` (tenant-aware) — never `redislib.NewRegistry` (single-tenant). `TestGetById_TenantIsolation` (Task 12) is the regression guard.
- **Wrong error classifier shape.** Use `errors.Is`, not type-switch. `TestClassifyError` (Task 8) is the regression guard.

The remaining risks (stale data on atlas-data redeploy, Redis outage, marshal cost) are operational — design.md §11 and PRD §1 cover them.
