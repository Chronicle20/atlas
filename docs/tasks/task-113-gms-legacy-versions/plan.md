# GMS Legacy Versions (48.1 / 61.1 / 72.1 / 79.1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring up GMS 48.1, 61.1, 72.1, and 79.1 as fully supported, runnable tenants — version-correct socket templates, operation registries, IDA exports, a comprehensively verified coverage-matrix column per version, audited version-gated Go branches, ingested WZ data, wired k8s ports, and a real-client end-to-end playthrough — all alongside the existing GMS 83/84/87/95 and JMS 185 tenants with zero regression.

**Architecture:** Four sequential **vertical passes** in descending order (79 → 72 → 61 → 48). Each pass runs the same nine-stage procedure (A–I): protocol layer (registry → template → IDA export → static audit → matrix + byte-fixture verification campaign), a code-gate audit slice, then runtime bring-up (WZ ingest → k8s ports + tenant provision → end-to-end playthrough). The just-completed higher version is the nearest-neighbor **anchor** for the next-lower version; the GMS 95 IDB/PDB is the tie-breaker. The IDB is the sole arbiter — "same as anchor" is a finding that requires switch-case evidence, never a default. A shared, accumulating `code-gate-audit.md` table (FR-7) is enumerated once and filled per pass; a final reconciliation pass closes the task.

**Tech Stack:** Go 1.x microservices (`services/atlas-*`, `libs/atlas-*`), `tools/packet-audit` CLI, IDA-Pro via ida-pro-mcp, atlas-configurations JSON seed templates, atlas-data WZ ingestion + object storage, Kubernetes (`deploy/k8s/base`), Kafka. Verification is byte-fixture tests with `packet-audit:verify` markers + the coverage matrix (`docs/packets/audits/STATUS.md`).

- **Worktree:** all work happens in the task-113 worktree (`<worktree-root>` = the `.worktrees/task-113-gms-legacy-versions` checkout) on branch `task-113-gms-legacy-versions`. - **No existing-version behavior change.** Adding the four versions must not alter GMS 83/84/87/95 or JMS 185. All version logic is explicit and version-scoped; no global flags. Regression-verify at least the v83 login→channel→map path each pass.
- **Verify, never invent.** Packet/opcode/operation/game-data values come from IDA source and WZ data only — never from general MapleStory memory. An unresolved fname is a **stop-and-ask**, never a fabricated or auto-substituted name (`feedback_unresolved_fname_escalate`). Quote actual tool output before drawing a conclusion.
- **No false passes.** A cell is verified only with byte-level evidence per `docs/packets/audits/VERIFYING_A_PACKET.md`. Dispatcher families need a discrete struct per mode + config-resolved mode byte + per-mode body + per-mode byte fixture (`docs/packets/DISPATCHER_FAMILY.md`); mode-byte enumeration is a false pass and is graded ❌ (`feedback_dispatcher_mode_byte_is_false_pass`).
- **No deferring producible work.** No `// TODO`, stubbed handlers, or 501s in landed commits. If the blocker is something you can produce (an unnamed IDB fn → name it; an unrouted template → wire it), produce it. - **IDB pre-flight before every read.** Confirm the loaded IDB instances/ports via `mcp__ida-pro__list_instances` (ports follow launch order — never hardcode; memory: `reference_ida_instance_ports_shifted_idbs_v9`). - **The IDA export splice is surgical and non-idempotent** — bootstrap from the descending anchor's export, purge cross-IDB coincidentals, never overwrite a committed export wholesale (`reference_packet_audit_serverbound_verification`).
- **Build/verification gate** (run from `<worktree-root>` for every pass that touches Go): `go test -race ./...`, `go vet ./...`, `go build ./...` clean in every changed module; `docker buildx bake atlas-<svc>` for every service whose `go.mod` was touched; `tools/redis-key-guard.sh` clean (run with `GOWORK=off`). ## Stage Procedure Reference (mandatory reading for every pass task)

This section is the DRY home for the nine-stage procedure. Treat this reference like Global Constraints: an engineer executing any stage task reads the matching procedure here in full. Commands are run from `<worktree-root>` unless stated. The canonical playbooks are `docs/packets/audits/STARTING_A_NEW_VERSION_PASS.md` (orchestration), `docs/packets/audits/VERIFYING_A_PACKET.md` (single cell), and `docs/packets/DISPATCHER_FAMILY.md` (dispatcher families).

### Stage A — Anchor & delta discovery
**Playbook:** PRD §4.1, IDA-MCP. **Output:** `docs/tasks/task-113-gms-legacy-versions/v<MAJOR>-packet-delta.md`.
1. IDB pre-flight: `mcp__ida-pro__list_instances`; confirm the `<MAJOR>` IDB by binary name; record its `<PORT>`.
2. From the `<ANCHOR>` opcode + operation maps, walk each in-scope dispatcher/handler in the `<MAJOR>` IDB via `func_query` `name_regex`. Cross-reference the GMS 95 IDB/PDB when the descending anchor is ambiguous.
3. Record in `v<MAJOR>-packet-delta.md`, with IDB function-name/address or anchor-confirmed evidence for **every** entry:
   - (a) inbound (handler) + outbound (writer) opcode map (FR-1.1);
   - (b) operation/mode (sub-op) table map — per-dispatcher mode bytes from the `<MAJOR>` dispatcher switch, each value citing its switch-case (FR-1.2);
   - (c) every packet whose structure/encoding differs from `<ANCHOR>` for the in-scope flows (login → channel → map → movement/chat plus every tier-1 packet) — exhaustive; for other flows, what was checked vs assumed (FR-1.3);
   - (d) "same as anchor" entries stated explicitly **with** evidence (FR-1.4);
   - (e) `usesPin` confirmed from the client (OQ-2; expected `false`);
   - (f) login-flow divergence vs `<ANCHOR>` (handshake, character-list encoding, world/channel list, character-select) — OQ-3, the biggest connect-risk, front-loaded here.
4. Enumerate the dispatcher families needing per-version operation-table extraction (OQ-7) — this sizes Stage E's campaign.
**Advance gate:** every in-scope opcode + mode value in `v<MAJOR>-packet-delta.md` cites IDB or anchor evidence; OQ-2/OQ-3 answered for this version.
